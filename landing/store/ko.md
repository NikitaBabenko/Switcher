# VibeNest Switcher: 상세 설명 (한국어)

`dkssudgktpdy` 라고 쳤는데 사실은 `안녕하세요` 였습니다. `Ctrl+Shift+L` 을 누르세요. 깨진 글자가 그 자리에서 올바른 텍스트로 바뀝니다. 다시 입력할 필요도, 필드를 떠날 필요도, 서버로 단 한 글자를 보낼 필요도 없습니다.

VibeNest Switcher는 잘못된 키보드 레이아웃으로 입력한 텍스트를 단축키 하나로 바로잡는 Chrome 확장입니다. 기본값은 완전 오프라인입니다. 소스 코드는 GitHub에 공개되어 있습니다. 12개 키보드 레이아웃 (영어, 러시아어, 우크라이나어, 벨라루스어, 독일어, 프랑스어, 그리스어, 히브리어, 터키어, 폴란드어, 스페인어, 한국어)과 12개 인터페이스 번역이 함께 제공됩니다.

============================================================

무엇이 다른가

100% 오픈소스. 텍스트를 다루는 모든 코드가 GitHub에 공개되어 있습니다: https://github.com/NikitaBabenko/Switcher. 로컬에서 테스트를 돌려 보고, 소스에서 빌드하고, 자신의 브라우저에서 검증할 수 있습니다.

기본적으로 완전 오프라인. 언어 모델이 확장 패키지 안에 들어 있으며 약 270 KB입니다. 텔레메트리 없음, 분석 없음, 클라우드 호출 없음, 서드파티 스크립트 없음. 변환은 절대 기기를 떠나지 않습니다.

계정 없음, 로그인 없음, 결제 없음, 광고 없음. 설치하고 바로 사용.

기본 12개 키보드 레이아웃. 대부분의 대안은 한두 쌍 (러시아어/영어) 만 지원합니다. VibeNest는 사용자가 활성화한 어떤 쌍이든 처리합니다.

최신 Manifest V3 빌드. 감사된 최소 권한, 2025년 Chrome 정책 준수. 사용자 쪽 마이그레이션 작업 없음.

크로스 브라우저. Chrome과 모든 Chromium 포크 (Edge, Brave, Opera, Vivaldi) 에서 동작합니다. Edge와 Firefox용 독립 빌드는 로드맵에 있습니다.

============================================================

기능

키보드 단축키 (Ctrl+Shift+L) 와 툴바 팝업 버튼. 단축키는 활성 텍스트 필드를 그 자리에서 다시 씁니다. 팝업은 원본과 수정본을 두 패널로 보여 주므로, 읽기 전용 필드나 적용 전에 변환을 확인하고 싶을 때 유용합니다.

붙여넣고 수정 모드. 잘못된 텍스트를 팝업에 붙여넣으면 수정된 버전이 나오고, 클릭 한 번으로 복사할 수 있습니다. 주소창, 내보내기 뷰, 읽기 전용 컨텍스트에서 이미 복사해 둔 텍스트에도 적용됩니다.

우클릭 컨텍스트 메뉴. 페이지에서 텍스트를 선택한 뒤 우클릭하고 "레이아웃 수정"을 선택하면, 마우스에서 손을 떼지 않고 처리할 수 있습니다.

입력 중 자동 수정. 옵션, 기본 꺼짐. 켜면 확장이 입력을 감시하고, 공백 뒤에 잘못된 레이아웃의 단어를 자동으로 바로잡습니다. 자동 수정 직후에 백스페이스를 누르면 그 수정이 즉시 거부됩니다. 비밀번호처럼 보이는 필드 (input type=password, autocomplete=current-password), OTP 코드, 카드 번호 필드는 자동 수정이 절대 건드리지 않도록 코드 차원에서 제외되어 있습니다.

페이지별 실행 취소. 팝업이 현재 페이지의 마지막 수정 한 번을 되돌릴 수 있도록 합니다. 변환이 잘못된 방향으로 갔을 때 (예: 키릴-라틴 사이의 고유명사 경계) 유용합니다.

사이트 어댑터. 비표준 입력을 다루는 사이트를 위한 핸들러가 미리 준비되어 있습니다 (contenteditable 래퍼, React로 관리되는 입력, 프레임 격리 컴포저): Twitter/X, Facebook, Messenger, VK, Instagram, 텔레그램 웹, 왓츠앱 웹, Discord, Slack, Reddit, LinkedIn, Twitch, Mastodon. 범용 어댑터가 나머지를 처리합니다.

사이트 정책. 특정 호스트 (은행, 인트라넷, 비밀번호 관리자) 를 차단하거나, 신뢰하는 호스트의 화이트리스트로만 확장을 제한할 수 있습니다. 호스트별 규칙은 세션을 넘어 유지됩니다.

스마트 언어 기본값. 확장을 처음 열면 브라우저의 로케일을 읽고 가장 가능성 높은 입력 언어를 미리 선택합니다. 옵션에서 언제든 변경할 수 있습니다. 감지는 활성화된 언어에 대해서만 동작하므로 결과가 빠르고 정확합니다.

기본 12개 키보드 레이아웃. 영어 (US QWERTY), 러시아어 (ЙЦУКЕН), 우크라이나어, 벨라루스어, 독일어 (QWERTZ), 프랑스어 (AZERTY), 그리스어, 히브리어, 터키어 (Q), 폴란드어 (214), 스페인어 (QWERTY), 한국어 (두벌식 / Dubeolsik).

12개 인터페이스 번역. 팝업과 옵션 페이지가 en, ru, uk, be, de, fr, el, he, tr, pl, es, ko로 현지화되어 있습니다. 히브리어는 오른쪽에서 왼쪽 방향을 따릅니다.

============================================================

동작 원리

감지 엔진은 활성화된 모든 레이아웃 쌍을 통해 각 문자를 전치한 다음, 문자 트라이그램 언어 모델이 양방향을 평가합니다. 모델은 각 언어에서 빈도 상위 3,000개 단어로 학습되었습니다. 자연스러운 텍스트로 더 높게 평가된 방향이 필드에 적용됩니다.

Caps Lock 휴리스틱과 "대소문자 자연스러움" 타이브레이커가 경계 사례를 처리합니다. `hELLO` 는 `Hello` 로, 의도된 `CAPS LOCK` 은 그대로, `JavaScript` 같은 단어 중간의 혼합 대소문자는 보존됩니다.

알고리즘은 의도적으로 LLM을 쓰지 않습니다. 전치 테이블과 작은 통계 스코어러뿐입니다. 그 덕분에 내장 언어 모델은 약 270 KB이고, 변환은 1밀리초 이하로 끝납니다. 활성 입력 필드에서 인라인으로 쓰기에 딱 맞는 프로파일입니다. 모델이 내장되고 고정되어 있으므로 모든 설치가 동일하게 동작합니다. 서버에서 몰래 추가 학습이 일어나는 일은 없습니다.

한국어는 엔진에서 가장 흥미로운 사례입니다. 화면의 모든 한글 음절이 호환 자모 키 입력 시퀀스로 가역적으로 분해되어, 같은 "전치하고 점수 매기기" 알고리즘이 그대로 동작합니다. 점수를 매긴 뒤 결과를 다시 한글 음절로 결합합니다.

누구를 위한 도구인가

- 한글과 영어를 하루에 여러 번 오가는 사용자.
- 출발 언어와 도착 언어 사이를 문단 한가운데서 넘나드는 번역가.
- 다국적 팀에서 일하는 개발자와 IT 담당자 (커밋 메시지, 티켓 댓글, 문화권을 넘는 코드 리뷰).
- 동료에게 "Уважаемые коллеги, ifkjvtt..." 로 시작하는 메일을 보내고 싶지 않은 헬프데스크 팀.
- 사전, 노트, 강사와의 채팅 사이에서 텍스트를 옮기는 어학 학습자와 강사.
- 멀티 스크립트 워크플로우에서 작업하는 모든 사람: 한글과 라틴, 키릴과 라틴, 그리스와 라틴, 히브리와 라틴.

개인정보 보호

기본값으로 확장은 오프라인에서 동작합니다. 내장 감지기는 브라우저 안에 머뭅니다. 텍스트도, 메타데이터도, 이벤트도 어디로도 전송되지 않습니다. 확장에는 분석 SDK, 텔레메트리, 원격 로깅, 서드파티 스크립트가 없습니다. 패키지는 GitHub에서 처음부터 끝까지 읽을 수 있을 만큼 작습니다.

권한과 그 사유는 개인정보 처리방침에 줄 단위로 기록되어 있습니다: https://vibenest.net/switcher/privacy

옵션에는 선택적 원격 API 폴백이 있지만 기본은 꺼짐, URL 필드는 비어 있습니다. 직접 켜지 않으면 텍스트 변환과 관련된 네트워크 호출은 단 한 번도 발생하지 않습니다.

자주 묻는 질문

Q: 잘못된 레이아웃으로 입력한 텍스트는 어떻게 고치나요?
A: 텍스트 필드에 커서를 두고 Ctrl+Shift+L을 누르세요. 확장이 어떤 레이아웃 쌍인지 감지하고, 문자를 전치하고, 필드를 그 자리에서 다시 씁니다. Twitter, Slack, Discord, 텔레그램 웹, 카카오톡 웹, Gmail, Notion, Reddit, LinkedIn 등 열린 웹의 거의 모든 텍스트 필드에서 동작합니다.

Q: 한영 전환은 어떻게 하나요?
A: VibeNest Switcher는 운영체제의 활성 입력 언어 전환을 대체하지 않습니다. 입력 언어 자체를 바꾸려면 평소처럼 한/영 키나 시스템 단축키를 그대로 쓰세요. 확장은 이미 잘못된 레이아웃으로 입력된 텍스트를 고치는 일에 집중합니다.

Q: 키릴을 라틴으로 음성 변환하나요?
A: 아닙니다. 이것은 레이아웃 수정이며 음성 입력이 아닙니다. 물리 자판이 없는 알파벳을 위한 음성 입력 도구는 다른 카테고리입니다.

Q: 오프라인에서 동작하나요?
A: 예. 기본값으로 완전 오프라인입니다. 트라이그램 모델은 확장 패키지 안에 있습니다.

Q: 어떤 권한이 필요하고 왜 필요한가요?
A: 브라우저 전반에서 텍스트를 다시 쓰기 위한 최소 권한: activeTab, scripting, storage, contextMenus. 각 권한은 PRIVACY.md에 사유와 함께 기록되어 있습니다.

Q: 입력 중 자동 수정을 어떻게 켜나요?
A: 확장 팝업을 열고 톱니바퀴 아이콘을 누른 다음 자동 수정을 켜세요. 자동 수정이 일어난 직후 백스페이스를 누르면 즉시 거부할 수 있습니다.

Q: Edge, Brave, Opera에서 동작하나요?
A: 예, 모든 Chromium 기반 브라우저에서. Edge는 Chrome 웹 스토어에서 MV3 빌드를 그대로 가져옵니다.

Q: 레이아웃 X (일본어, 힌디어, 체코어) 를 추가할 수 있나요?
A: 예, 요청해 주세요. 레이아웃 추가는 파일 두 개 (46자 표 + 빈도 상위 3,000 단어 목록) + 모델 재빌드면 됩니다.

새로운 변경 사항

버전 1.0.1 (2026년 5월 13일)
- 인터페이스 문자열과 랜딩 페이지 텍스트에서 긴 대시를 제거하여 일관된 타이포그래피 유지.

버전 1.0.0 (2026년 5월 12일)
- 안정 릴리스. 옵션에 숨겨진 원격 API 폴백은 기본 꺼짐, URL 필드는 비어 있음. 기본 설치는 네트워크에 접속하지 않음.

이전 릴리스에서는 한국어, 폴란드어, 스페인어 레이아웃이 추가되었고 (v0.3.0), Twitch와 Mastodon용 사이트 어댑터, 페이지별 실행 취소 동작이 도입되었습니다.
