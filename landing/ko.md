# 키보드 레이아웃 전환기: 12개 언어로 한영 전환 오타 수정

`dkssudgktpdy` 라고 쳤는데 사실은 `안녕하세요` 라고 쓰고 싶었던 적이 있나요. **VibeNest Switcher**는 잘못된 키보드 레이아웃으로 입력한 텍스트를 단축키 한 번으로 바로잡는 Chrome 확장입니다. 선택, 복사, 다시 입력이 필요 없습니다. 입력 필드에서 커서만 두고 **`Ctrl+Shift+L`** 만 누르면 그 자리에서 다시 쓰입니다. 감지는 모두 오프라인으로 돌아가고, 소스는 GitHub에 공개되어 있으며, 한글 두벌식을 포함한 12개 키보드 레이아웃이 기본 제공됩니다.

## 잘못된 키보드 레이아웃을 단축키 하나로 고치기

매일 한글과 영어를 오가며 글을 쓰는 사람이라면 그 리듬을 압니다. 한영 전환을 잊고 입력하고, 화면을 보고, 선택하고, 지우고, 다시 입력합니다. 운영체제의 키보드 레이아웃 전환은 *다음* 글자에만 영향을 줄 뿐, 이미 화면에 떠 있는 것은 고치지 않습니다. VibeNest Switcher가 바로 그 빠진 버튼입니다. 텍스트 필드에 커서를 두고 단축키를 누르면, 그 자리에서 올바른 레이아웃으로 다시 쓰입니다. Twitter/X, Slack, Discord, 텔레그램 웹, 카카오톡 웹, Gmail, Notion, LinkedIn, Reddit, 거의 모든 웹 입력 필드에서 동일하게 동작합니다.

### 전 / 후

| 입력한 것 (잘못된 레이아웃) | 의도한 것 |
|---|---|
| `dkssudgktpdy` | `안녕하세요` |
| `안세ㅣㅐ` | `Hello` |
| `gngnkrh` | `후후하고` |
| `ghbdtn` | `привет` |
| `xfq` | `чай` |

같은 단축키가 12개 지원 레이아웃 사이에서 양방향으로 작동합니다. 잘못된 텍스트를 툴바 팝업에 붙여넣고, 수정된 버전을 클릭 한 번으로 복사해 가져갈 수도 있습니다.

## 아무도 이야기하지 않는 잘못된 레이아웃 문제

한영을 오가는 사람, 한국어와 영어를 오가는 개발자나 디자이너, 다국적 팀의 IT 담당자, 해외 자료를 옮겨 쓰는 학생과 연구자, 한국어와 일본어 또는 한국어와 중국어를 함께 쓰는 사람: 모두 이 작은 비용을 압니다. 한 번에 몇 초씩, 하루에 수십 번, 일 년이면 손과 운영체제 사이의 나쁜 습관 때문에 며칠 분량의 시간을 잃습니다.

운영체제의 한영 전환 키는 이를 해결하지 못합니다. 활성 입력 언어를 바꿔주기는 해도 이미 입력된 텍스트는 고치지 않습니다. 음성 변환 도구 (`privet` 를 `привет` 로 바꾸는 식) 역시 이 문제를 풀지 않습니다. 그것은 키릴 자판이 없는 사람이 키릴로 입력하기 위한 도구로, 전혀 다른 카테고리입니다. **VibeNest Switcher는 바로 그 순간을 위한 것입니다.** 입력을 끝낸 뒤 레이아웃이 잘못되었음을 알아채고, 필드에서 벗어나지 않은 채 한 동작으로 고치고 싶은 그 순간입니다.

데스크톱에서는 오래전부터 작은 레이아웃 전환 유틸리티 가족이 답이었습니다. 모두 폐쇄 소스, 모두 Windows 전용, 그리고 오늘날 실제로 입력이 일어나는 현대 웹 앱 안으로는 손이 닿지 않습니다. VibeNest Switcher는 이 영역을 채우는 오픈소스, 브라우저 네이티브, 완전 오프라인 대안입니다. 입력이 실제로 발생하는 필드 안에서, Chrome 또는 Chromium이 도는 모든 운영체제에서 동작합니다.

## 기능

- **단축키** (`Ctrl+Shift+L`) 와 툴바 팝업 버튼으로 활성 입력 필드를 즉시 수정
- **붙여넣고 수정** 모드: 잘못된 텍스트를 팝업에 붙여넣으면 수정된 버전이 나오고, 클릭 한 번으로 복사
- **우클릭 컨텍스트 메뉴**로 선택한 텍스트를 마우스에서 손 떼지 않고 수정
- **입력 중 자동 수정**: 옵션, 기본은 꺼짐. 백스페이스로 자동 수정을 즉시 거부
- **페이지별 실행 취소**: 현재 페이지의 마지막 수정 한 번을 클릭 하나로 되돌리기
- **사이트 어댑터**: Twitter/X, Facebook, Messenger, VK, Instagram, 텔레그램 웹, 왓츠앱 웹, Discord, Slack, Reddit, LinkedIn, Twitch, Mastodon, 그리고 나머지를 다루는 범용 어댑터
- **사이트 정책**: 특정 호스트 (은행, 인트라넷) 차단, 또는 신뢰할 수 있는 사이트의 화이트리스트로만 제한
- **스마트 언어 기본값**: 처음 설치 시 브라우저 로케일을 읽어 입력 언어를 자동 선택하고, 옵션에서 언제든 변경
- **기본 제공 12개 키보드 레이아웃**: 영어 (US QWERTY), 러시아어 (ЙЦУКЕН), 우크라이나어, 벨라루스어, 독일어 (QWERTZ), 프랑스어 (AZERTY), 그리스어, 히브리어, 터키어 (Q), 폴란드어 (214), 스페인어 (QWERTY), 한국어 (두벌식). 한국어는 특별 처리됩니다. 엔진이 한글 음절을 호환 자모로 분해하고, 전치한 뒤 다시 한글로 결합합니다
- **12개 인터페이스 번역**: en, ru, uk, be, de, fr, el, he, tr, pl, es, ko

Manifest V3 빌드, 감사된 최소 권한, 2025년 Chrome 정책에 마이그레이션 없이 바로 대응됩니다.

## 동작 원리

감지 엔진은 활성화된 모든 레이아웃 쌍을 통해 각 문자를 전치한 다음, 언어별 빈도 상위 3,000 단어로 학습된 문자 트라이그램 모델로 양방향을 평가하고, 자연스러운 텍스트로 보이는 쪽을 유지합니다. Caps Lock 휴리스틱과 "대소문자 자연스러움" 타이브레이커가 경계 사례를 처리합니다 (`hELLO` 는 `Hello` 로, 의도된 대문자 입력은 그대로). 이 모두가 브라우저 안에서 일어납니다. 원격 모델로 가는 데이터, API 호출, 네트워크 지연이 모두 없습니다.

알고리즘은 **LLM 기반이 아닙니다**. 전치 테이블과 작은 통계 스코어러뿐입니다. 그래서 내장 언어 모델은 약 270 KB이며, 변환은 1밀리초 이하로 끝납니다. 입력 필드 안에서 인라인으로 쓰기에 딱 맞는 프로파일입니다. 모델이 내장되어 있고 고정되어 있으니 모든 설치가 같은 방식으로 동작합니다. 서버에서 몰래 추가 학습이 일어나는 일은 없습니다.

## 자주 묻는 질문

### 잘못된 키보드 레이아웃으로 입력한 텍스트는 어떻게 고치나요?

VibeNest Switcher를 설치하고, 잘못 입력된 텍스트가 들어 있는 필드에 커서를 두고 **`Ctrl+Shift+L`** 을 누르세요. 확장이 어떤 레이아웃 쌍에서 입력되었는지 감지하고, 문자를 전치하고, 필드를 그 자리에서 다시 씁니다. 선택하거나 복사하거나 다시 입력할 필요가 없습니다. Twitter, Slack, 텔레그램 웹, 카카오톡 웹, Discord, Gmail, Notion, Reddit, LinkedIn 등 모든 웹 입력 필드에서 같은 단축키가 동작합니다. 필드를 직접 수정할 수 없는 경우 (예: 읽기 전용 내보내기 뷰)에는 수정된 텍스트를 클립보드에 복사하고 작은 알림으로 알려 줍니다.

### 한영 전환은 어떻게 하나요?

VibeNest Switcher는 운영체제의 활성 입력 언어 전환을 대체하지 않습니다. 입력 언어 자체를 바꾸려면 평소처럼 한/영 키나 시스템 단축키를 그대로 쓰세요. 확장은 다른 문제를 푸는 도구입니다. 이미 잘못된 레이아웃으로 입력한 텍스트를 고치는 일입니다. 영어로 쓰려 했는데 한글로 입력되었다면 (`dkssudgktpdy` 가 화면에 떠 있다면), 단축키 한 번으로 그 자리에서 다시 쓰입니다. 반대 방향도 똑같습니다.

### 다국어 키보드 도구인가요?

예. VibeNest Switcher는 12개 레이아웃 (영어, 러시아어, 우크라이나어, 벨라루스어, 독일어, 프랑스어, 그리스어, 히브리어, 터키어, 폴란드어, 스페인어, 한국어)을 지원하는 다국어 키보드 레이아웃 수정 도구이며, 인터페이스 번역도 12개 모두 포함됩니다. 단일 쌍 도구와 달리 VibeNest는 활성화된 어떤 쌍 사이에서도 감지하고 변환합니다. 실제로 쓰는 언어만 켜 두세요. 감지가 그 집합에 집중되므로 결과가 더 빠르고 정확해집니다. 매일 세 가지 이상의 알파벳을 쓴다면 단일 쌍 도구 대비 의미 있는 향상입니다.

### Punto Switcher와 어떻게 다른가요?

그것은 Windows용 데스크톱 앱입니다. VibeNest Switcher는 Chrome과 모든 Chromium 브라우저 안에서 완전히 동작하는 독립 오픈소스 대안이므로, macOS, Linux, ChromeOS, Windows에서 똑같이 돌아갑니다. 그리고 오늘날 실제 입력이 일어나는 웹 필드 안에서 동작합니다. 감지 아이디어는 같습니다 (전치하고 점수 매기기). 하지만 모델은 확장 안에 들어 있고 오프라인에서 동작합니다. 텍스트의 단 한 글자도 브라우저를 떠나지 않습니다. 소스는 [GitHub](https://github.com/NikitaBabenko/Switcher)에 공개되어 있어 개인정보 보호 주장은 코드로 검증할 수 있습니다.

### 음성 변환을 하나요?

아니오. 이것은 레이아웃 수정이지 음성 입력이 아닙니다. 키릴 자판이 없을 때 `privet` 으로 `привет` 를 입력하는 도구를 찾는다면 그것은 다른 카테고리 (Cyrillatin, Translit)입니다. VibeNest Switcher는 양쪽 레이아웃이 이미 있는데 실수로 잘못된 쪽에서 입력했고, 결과를 그 자리에서 고치고 싶을 때를 위한 것입니다.

### 오프라인으로 동작하나요? 어떤 데이터가 브라우저를 떠나나요?

기본적으로 완전 오프라인입니다. 내장 트라이그램 언어 모델은 확장 패키지 안에 들어 있고, 감지는 사용자 기기에서 실행됩니다. **텍스트와 메타데이터 어느 것도 어디로도 전송되지 않습니다.** 분석 SDK 없음, 텔레메트리 없음, 원격 로깅 없음, 서드파티 스크립트 없음. 옵션에 원격 API 폴백이 있지만 **기본은 꺼져 있고** URL 필드도 비어 있습니다. 직접 켜지 않으면 텍스트 변환과 관련된 네트워크 호출은 단 한 건도 발생하지 않습니다. 권한과 그 사유는 [PRIVACY.md](https://vibenest.net/switcher/privacy)에 줄 단위로 적혀 있고, 데이터 흐름 경로는 GitHub의 소스로 감사할 수 있습니다.

### 어떤 12개 키보드 레이아웃이 지원되나요?

영어 (US QWERTY), 러시아어 (ЙЦУКЕН), 우크라이나어, 벨라루스어, 독일어 (QWERTZ), 프랑스어 (AZERTY), 그리스어, 히브리어, 터키어 (Q), 폴란드어 (214), 스페인어 (QWERTY), 한국어 (두벌식 / Dubeolsik). 활성화된 어떤 쌍이든 감지에 사용할 수 있습니다. 한국어가 가장 흥미로운 사례입니다. 화면의 모든 한글 음절이 자모 시퀀스로 가역적으로 분해되어, 같은 전치-스코어 알고리즘이 그대로 동작합니다. 다른 레이아웃 (일본어, 힌디어, 체코어) 이 필요하면 알려 주세요. 한 레이아웃 추가는 파일 두 개 (46자 표 + 빈도 상위 3,000 단어 목록)면 되고, 검증되는 대로 추가합니다. GitHub에 이슈를 열거나 **info@vibenest.net** 으로 메일을 보내 주세요.

### 무료인가요? 오픈소스인가요?

두 가지 다 예. VibeNest Switcher는 무료입니다. 계정 없음, 결제 없음, 광고 없음. 전체 소스는 GitHub <https://github.com/NikitaBabenko/Switcher> 에 관용적 오픈소스 라이선스로 공개되어 있습니다. 텍스트를 다루는 모든 코드를 한 줄 한 줄 읽을 수 있습니다. 그것이 개인정보 보호 약속의 진짜 뜻입니다. 선언이 아니라 검증 가능. 버그를 발견하거나 새 기능을 원한다면 이슈 트래커가 시작점입니다.
